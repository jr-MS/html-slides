Why did it get dramatically better? Because I started treating it like an engineering project.

The experience did not improve because I found one magical prompt. It improved because I kept tuning memory, sessions, skills, schedulers, Notion, GitHub, permissions, and execution receipts.

What I changed underneath

  • Added long-term memory and "soul" rules
  • Fixed authentication across Notion, GitHub, Tavily, and Azure
  • Turned "reset the session" and "remember this" into command phrases
  • Changed external write-backs so success is only reported after the action really happened

What changed because of that

The need is often small even when the technology is not. Start with one tiny problem you truly want solved, and the tuning process will force you to learn a lot more.

The 4 feelings I most want to leave with the team

This is not a grand best-practices guide. It is the four-part takeaway I most want to keep after learning the hard way as a real user.

  1. The need is limited; the technology may not be. Start with one small, real problem. AI will push you toward new edges as you keep tuning.
  2. You have to accept failure and mess. I spent a long time tuning the diary flow, for example — it forgot, drifted, and ignored the format.
  3. Use the company's GitHub Copilot resources fully. The models are current and the surface area is wide. It is a great test field.
  4. Do not push everything through one entry point. One context gets messy fast when it carries too many roles. Split by role into separate agents or chats.
I believe this more every week: the best way to use AI is not to ask more cleverly, but to grow small needs into stable systems.
