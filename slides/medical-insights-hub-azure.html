Architecture Flow

12 步架构流程 — Azure 实现

完全对标 AWS Medical Insights Hub 原始架构的每一步，以 Azure 原生服务实现

1
用户通过 Azure Front Door 和 Azure Static Web Apps 访问用户界面控制台
用户/临床医生/分析师通过 Azure Front Door（集成 WAF 和全球 CDN）访问托管在 Azure Static Web Apps 上的前端界面。Front Door 提供全局负载均衡、DDoS 防护和自定义域名支持。
Azure Front Door Static Web Apps
2
应用程序使用 Microsoft Entra External ID（或 Entra ID B2C）进行身份验证
前端通过 OIDC/OAuth 2.0 协议与 Microsoft Entra External ID 进行交互，实现用户注册、登录、多因子认证和条件访问策略。支持社交身份提供商集成及自定义用户流。
Microsoft Entra External ID Entra ID B2C
3
应用程序访问通过 Azure API Management 提供的 RESTful API
已认证的请求经由 Azure API Management (APIM) 路由。APIM 提供策略引擎用于速率限制、JWT 验证、请求/响应转换、API 版本管理和开发者门户。
Azure API Management
4
Azure API Management 调用 Azure Functions
APIM 将验证过的 API 调用转发至 Azure Functions。Functions 以事件驱动的无服务器方式执行业务逻辑，支持 HTTP 触发器和 Durable Functions 编排。
API Management Azure Functions
5
Azure Functions 将持久性数据存储在 Azure SQL Database 中，将临时数据存储在 Azure Blob Storage 中
持久性的应用数据（用户数据、交易记录）存入 Azure SQL Database 或 Azure Database for PostgreSQL；临时数据和中间文件使用 Azure Blob Storage，支持热/冷/归档分层。
Azure SQL Database Azure Blob Storage
6
Azure Functions 从 Azure Key Vault 获取存储的 API 密钥，并调用 Insights Core
Functions 通过 Managed Identity 无密码访问 Azure Key Vault，获取 API 密钥、连接字符串和证书。使用 Key Vault References 实现零代码密钥轮换。
Azure Key Vault Azure Functions
7
Azure Container Apps 获取 API 密钥，并通过 Azure Functions 从外部源获取数据。临时数据存储在 Blob Storage 和 Cosmos DB 中
Azure Container Apps（替代 ECS）运行长时间运行的数据采集任务和批处理作业。Container Apps Jobs 用于一次性任务。所有临时数据存入 Blob Storage，工作流元数据和状态记录在 Cosmos DB 中。
Container Apps Blob Storage Cosmos DB
8
Azure Container Apps 派发任务，并将消息推送到 Azure Service Bus
容器任务完成后，将处理结果和任务消息发送到 Azure Service Bus 队列。Service Bus 提供 FIFO、死信队列(DLQ)、会话和事务支持，确保消息可靠传递。
Container Apps Azure Service Bus
9
Azure Service Bus 触发 Azure Functions 执行
Service Bus 队列中的新消息通过 Service Bus Trigger 自动触发 Azure Functions。Functions 处理每条消息并执行后续的推理和翻译流程。支持批量处理和并发控制。
Service Bus Trigger Azure Functions
10
Azure Functions 调用托管在 Azure Machine Learning 上的模型进行推理，翻译任务调用 Azure AI Translator
Functions 调用 Azure ML Managed Online Endpoints 进行模型推理（或通过 ACA/AKS 自托管模型服务）。翻译任务使用 Azure AI Translator 进行医学术语的多语言翻译。
Azure Machine Learning ACA / AKS Azure AI Translator
11
Azure Functions 在推理完成后通知 Azure Event Grid
推理和翻译完成后，Azure Functions 向 Event Grid Topic 发布完成事件。Event Grid 以推送方式将事件分发给所有订阅者，支持过滤、重试和死信。
Azure Functions Azure Event Grid
12
Azure Event Grid 触发 Azure Functions 执行，并返回推理结果
Event Grid 订阅触发另一组 Azure Functions，将推理结果写入数据库并通过 SignalR / WebSocket 推送给前端用户。完成端到端的医学洞察分析闭环。
Event Grid Azure Functions Azure SQL / Cosmos DB
