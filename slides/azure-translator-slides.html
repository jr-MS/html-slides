核心功能一览

02
💬

文本翻译 (Text Translation)

实时翻译文本字符串，支持 130+ 语言对，单次最大 50,000 字符。内置语言检测、音译和词典查询。

📄

文档翻译 (Document Translation)

批量翻译整个文档（PDF、DOCX、PPTX 等），保留原始格式和排版。支持异步批量和同步单文件模式。

🎯

Custom Translator

上传行业平行语料训练专属 NMT 模型。支持 Training / Dictionary / Tuning 三类文档。

⚡

Adaptive CT (Preview)

调用时传入上下文参考句对，模型实时自适应。无需训练，即时生效，按需付费。

🗣️

语音翻译 (Speech Translation)

与 Azure Speech 集成，实现实时语音到语音翻译。适用于会议、客服、直播等场景。

📦

容器化部署

Docker 容器离线运行翻译引擎。连接 / 断开两种模式，满足政府、金融等高安全需求。

💡 2025 Preview API 新增：LLM 翻译引擎（GPT-4o 驱动）、性别/语气控制、AdaptCT 自适应翻译

Python 代码示例

05
文本翻译 — 基础调用
Python import requests, uuid, json key = "YOUR_KEY" endpoint = "https://api.cognitive.microsofttranslator.com" headers = { "Ocp-Apim-Subscription-Key": key, "Ocp-Apim-Subscription-Region": "eastasia", "Content-type": "application/json", "X-ClientTraceId": str(uuid.uuid4()) } body = [{"text": "患者血压偏高，建议调整用药方案。"}] resp = requests.post( endpoint + "/translate?api-version=3.0&to=en", headers=headers, json=body ) print(json.dumps(resp.json(), indent=2))
使用 Custom Translator 模型
Python # 指定 category 参数使用自定义模型 category_id = "YOUR_CATEGORY_ID" resp = requests.post( endpoint + "/translate", params={ "api-version": "3.0", "to": "en", "category": category_id }, headers=headers, json=[{"text": "临床试验数据符合预期。"}] )
Dictionary Lookup 示例
Python resp = requests.post( endpoint + "/dictionary/lookup", params={"api-version": "3.0", "from": "zh-Hans", "to": "en"}, headers=headers, json=[{"text": "药物"}] ) # 返回候选翻译：drug, medicine, medication...

文档翻译

06
两种翻译模式

📦 异步批量翻译

  • 从 Azure Blob 批量读取文档 → 翻译 → 写回 Blob
  • 适合大量文档定期翻译场景
  • 单文件 ≤ 40 MB，单次请求总量 ≤ 250 MB
  • 支持长时间运行，API 返回操作 ID 轮询状态

⚡ 同步单文件翻译

  • 直接 POST 文件内容 → 返回翻译后文件
  • 适合实时按需翻译，无需 Blob 存储
  • 文件 ≤ 20 MB
支持文档格式
类型格式
OfficeDOCX, PPTX, XLSX
PDF标准 PDF（扫描件需先 OCR）
文本TXT, CSV, TSV, Markdown
本地化XLIFF (.xlf), HTML, MHTML
其他MSG (Outlook), SRT/VTT (字幕)
✅ 文档翻译 保留原始排版和格式，输出文件与源文件结构一致。表格、图表、页眉页脚均保留。
🔗 批量翻译中 "总量 ≤ 250 MB" 指一次批量请求中所有源文档的大小之和。

Custom Translator 自定义翻译

07
训练流程
📂 上传平行语料
→
⚙️ 训练模型
→
🚀 发布部署
→
🔗 API 调用
三种文档类型

📚 Training 训练文档

大量平行语料（源+目标），模型从中学习领域翻译模式。建议 ≥ 10,000 对齐句对。

📖 Dictionary 词典文档

强制精确翻译。模型必须按词典映射翻译指定术语。适合专有名词、品牌名等。
文件限制 100 MB（约 200-500 万对）。

📝 Tuning 调优文档

验证集，训练过程中用于评估模型质量、防止过拟合。建议 2,500 句对。

Training vs Dictionary
对比项TrainingDictionary
作用学习翻译风格强制精确映射
灵活性模型自主推断逐字匹配
数据量≥ 10K 句对不限（100MB）
适用行业通用语料术语、名词
💡 医疗场景推荐：同时上传 Training（医学论文、临床报告平行语料）+ Dictionary（药物名、疾病名精确映射）。一个项目中可同时包含两种文档类型。
📌 Custom Translator 训练的是 Text Translation 引擎。Document Translation 底层也调用 Text Translation 引擎，所以自定义模型在文档翻译中同样生效。

Adaptive Custom Translation (Preview)

08
什么是 AdaptCT？
  • 无需训练 — 在 API 请求中直接传入参考句对
  • 模型实时自适应，翻译风格向参考对齐
  • 适合 快速迭代 场景，术语随时可更新
  • 按需计费，无训练成本
  • Preview API：2025-05-01-preview
API 调用示例
JSON { "translationContext": { "references": [ { "source": "药物不良反应", "target": "Adverse Drug Reaction" }, { "source": "临床试验", "target": "Clinical Trial" } ] } }
AdaptCT vs Custom Translator

⚡ AdaptCT

  • 无需训练，即时生效
  • 参考句对随请求传入
  • 灵活性极高
  • 按请求计费
  • 适合术语频繁变化场景
  • Preview 阶段
VS

🎯 Custom Translator

  • 需要训练（数小时-天）
  • 模型持久化部署
  • 翻译质量更稳定
  • 训练免费 + 按请求计费
  • 适合稳定领域模型
  • GA 正式版
🔑 选择建议：术语固定且语料丰富 → Custom Translator；术语变化频繁或数据不足 → AdaptCT；两者可组合使用。

容器化部署

11
部署模式

☁️ Connected 连接模式

  • 容器在本地运行翻译，定期向 Azure 发送计量数据
  • 需要网络连接用于授权和计费
  • 翻译数据不出本地

🔒 Disconnected 断开模式

  • 完全离线运行，无需任何网络连接
  • 需要提前申请并签署承诺计划
  • 适合高安全/涉密环境（军事、政府）

安全合规与隐私

12
🚫

No-Trace 策略

翻译文本不被 Microsoft 留存、记录或用于改进模型。提交的数据仅用于实时翻译并立即丢弃。

🔒

传输与存储加密

TLS 1.2+ 传输加密，静态数据 AES-256 加密，支持客户管理密钥 (CMK)。

🏥

HIPAA 合规

Azure 平台提供 BAA 协议支持。配合容器部署 + Private Endpoint 满足医疗合规要求。

🌐

Private Endpoint

通过 VNet 私有链接访问 Translator API，数据完全不经过公网。

📦

容器隔离

在客户自有基础设施上运行容器，数据完全不离开客户环境，最高级别数据主权。

🛡️

PII 保护

配合 Azure AI Language 的 PII Detection 服务，先脱敏再翻译，保护患者/客户个人信息。

✅ Azure Translator 满足 SOC 2 Type 2、ISO 27001、HIPAA (via BAA)、GDPR、FedRAMP 等主要合规标准。
