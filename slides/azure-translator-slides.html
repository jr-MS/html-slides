核心功能矩阵

02
💬

文本翻译 (Text Translation)

  • 实时翻译，单请求支持多目标语言
  • 自动语言检测 / 音译 / 句子边界检测
  • 词典查找 & 上下文示例
  • 动态词典 & 防止翻译标记
  • Preview：LLM 翻译 (GPT-4o) + 自适应定制
  • 每请求最大 50,000 字符，数组最大 1,000 元素
📄

文档翻译 (Document Translation)

  • 异步批量：多文件并行翻译，保留原始格式
  • 同步单文件：直接返回翻译文件，无需 Blob Storage
  • 支持 Word / Excel / PDF / HTML / PowerPoint 等
  • 扫描 PDF & 图片 OCR 翻译
  • 术语表 (Glossary) 自定义
  • 批量限制：单文件 ≤40MB，总量 ≤250MB
🎯

自定义翻译 (Custom Translator)

  • 上传平行语料训练领域 NMT 模型
  • 需要 10,000+ 句对，训练周期 1-数天
  • BLEU 评分提升 5-10 点
  • 通过 API v3 category 参数无缝调用
  • 适合高频、大量、稳定的领域内容
  • 支持团队协作工作空间
⚡

自适应翻译 (Adaptive CT) NEW

  • 基于 LLM + few-shot learning 的新范式
  • 仅需 5-500 句对即可生效
  • 分钟级索引创建，无需重新训练
  • 适合快速变化、低量的动态内容
  • 通过 dataset index ID 调用
  • 与 GPT-4o / GPT-4o-mini 结合

代码示例 · Python 文本翻译

05
Python REST API 调用
Python import requests, uuid, json # Azure Translator 配置 key = "<YOUR-TRANSLATOR-KEY>" endpoint = "https://api.cognitive.microsofttranslator.com" location = "<YOUR-RESOURCE-LOCATION>" params = { 'api-version': '3.0', 'from': 'en', 'to': ['zh-Hans', 'ja'] # 多目标语言 } headers = { 'Ocp-Apim-Subscription-Key': key, 'Ocp-Apim-Subscription-Region': location, 'Content-type': 'application/json', 'X-ClientTraceId': str(uuid.uuid4()) } body = [{'text': 'Hello, friend!'}] resp = requests.post( endpoint + '/translate', params=params, headers=headers, json=body ) print(json.dumps(resp.json(), ensure_ascii=False, indent=2))
响应示例
JSON Response [{ "detectedLanguage": { "language": "en", "score": 1.0 }, "translations": [ { "text": "你好，朋友！", "to": "zh-Hans" }, { "text": "こんにちは、友達！", "to": "ja" } ] }]
cURL 快速示例
Bash curl -X POST \ "https://api.cognitive.microsofttranslator.com\ /translate?api-version=3.0&from=en&to=de" \ -H "Ocp-Apim-Subscription-Key: $KEY" \ -H "Content-Type: application/json" \ -d '[{"text":"How much for coffee?"}]'
✅ 一次请求多语言 — to 参数支持数组，一次调用获取多种语言翻译

Custom Translator · 领域专属翻译

07
🎯 用行业术语和领域数据训练定制化 NMT 模型，获得比通用翻译 高 5-10 个 BLEU 分 的翻译质量
为什么需要自定义翻译？
  • 通用翻译无法理解行业专属术语（医药 / 法律 / 制造业）
  • 确保品牌名称、产品名的一致翻译
  • 遵循企业特定的翻译风格和语气
  • NMT 全句上下文理解，提供更流畅的人工级翻译
关键特性
  • 支持 ALIGN / PDF / TMX / XLIFF / DOCX / XLSX 等格式
  • 文档级平行数据自动匹配
  • 可用纯词典数据训练（Dictionary-only）
  • 团队协作工作空间 & BLEU 评分评估
  • 训练部署无需编程，通过 Portal 完成
训练流程
① 创建工作空间 (Workspace)
↓
② 上传平行语料 (Training / Tuning / Testing)
↓
③ 训练 NMT 模型 (1-数天)
↓
④ BLEU 评分评估 (Evaluate)
↓
⑤ 部署 & 通过 category 参数调用
调用方式
REST API # 添加 category 参数使用自定义模型 GET /translate?api-version=3.0 &from=en&to=de &category=<your-category-id>
⚡

Adaptive Custom Translation (AdaptCT)

08
🚀 全新范式：不再从头训练模型，而是利用现有 LLM (GPT-4o) + 小数据集 + 智能索引，实现即时领域翻译定制。"把法拉利调校为适合你赛道的赛车"
核心优势

⚡ 极速上线

仅需 5-500 句对即可创建索引，分钟级生效。传统方式需 10,000+ 句对 + 24-48小时训练

🔄 即时更新

更新数据集 → 重建索引 → 立即生效。无需重训练、无需重部署，实现翻译的持续迭代

🎯 Few-Shot Learning

提交翻译请求时，系统检索最相关句对作为 few-shot 示例，引导 LLM 生成领域精准翻译

工作流程
① 创建工作空间 & 上传句对 (TMX/TSV)
↓
② 创建 Dataset Index（分钟级）
↓
③ 调用 Translate API + index ID
↓
④ LLM few-shot 检索 → 精准翻译
数据格式示例 (TMX)
TMX <tmx version="1.4"> <body> <tu> <tuv xml:lang="en"> <seg>Click the submit button</seg> </tuv> <tuv xml:lang="fr"> <seg>Cliquez sur soumettre</seg> </tuv> </tu> </body> </tmx>

AdaptCT vs Custom Translator · 如何选择

09
维度⚡ Adaptive CT (AdaptCT)🎯 Custom Translator
数据量要求5 - 10,000 句对10,000+ 句对
上线速度分钟级（索引创建）1-数天（模型训练）
更新方式更新数据集 → 重建索引 → 即刻生效重新训练 → 重新部署
技术原理LLM + Few-Shot Learning + 索引检索NMT 模型全量训练
维护成本零维护（上传 → 索引 → 调用）需管理训练、调优、测试数据集
适用内容动态、快速变化、低频内容稳定、高频、大量内容
典型场景客服工单 / 产品文档 / 动态内容法律合同 / 医学报告 / 标准规程
API 调用Preview API + dataset index IDv3 API + category 参数
⚡
选 AdaptCT： 数据量少 (< 10K) · 需快速迭代 · 动态内容 · 客服/SaaS/电商 · 快速原型验证
🎯
选 Custom Translator： 数据量充足 (> 10K) · 内容稳定 · 严格术语控制 · 法律/医药/制造
AdaptCT 最佳实践

🎯 从小开始

先用 50-100 高价值句对测试效果，再逐步扩展

📂 按领域分索引

customer-support-index、product-docs-index 分域管理

✨ 质量 > 数量

5 个完美句对比 500 个平庸句对更有效，聚焦高影响术语
