定制化原语 — 四层结构

.github/ ├── copilot-instructions.md ← 全局指令 ├── instructions/ │ └── *.instructions.md ← applyTo ├── prompts/ │ └── *.prompt.md ← / 命令 ├── agents/ │ └── *.agent.md ← 菜单选择 └── skills/ └── SKILL.md ← 自动发现

📋 Instructions — 上下文工程

---
applyTo: "**/*.py"
---
# Documentation Standards
## Docstring Format
- Use Google-style docstrings
- Always include Args, Returns, Raises
## Constraints
- Reference existing codebase patterns
- NEVER use NumPy-style docstrings
Glob 模式匹配
**/*.py所有 Python
**/*.{js,ts}所有 JS/TS
**/*.test.*所有测试文件
src/**/*.tsxsrc 下 TSX
💡 保持指令简洁可行动 — 过长的指令会降低效果

🤖 Agents — 多 Agent 编排

核心模式：分析器 → Handoff → 生成器

📖
Analyzer
只读分析
🔄
Handoff
传递结果
✍️
Generator
生成代码
---
name: Doc Analyzer
description: Analyze code for documentation needs
tools: ['search', 'usages', 'fetch']
handoffs:
  - label: "Write Documentation"
    agent: doc-writer
    prompt: "Based on analysis, generate docs."
---
